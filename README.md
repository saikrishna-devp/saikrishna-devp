## 🚀 Featured Projects

> 💡 Each project below mirrors real-world enterprise data engineering problems I've solved professionally.

### 🏥 [Healthcare Claims ETL Pipeline](https://github.com/saikrishna-suryavamsham/healthcare-claims-etl)
> End-to-end pipeline processing CMS Medicare claims data into a Snowflake data warehouse

**Tech:** Python · PySpark · Apache Airflow · AWS S3 · Snowflake · Great Expectations

- Processes **5M+ records** through a multi-stage ETL pipeline (Raw → Processed → Curated)
- Airflow DAGs with retry logic and failure alerting achieving **99.9% uptime**
- Star-schema dimensional model in Snowflake optimized for claims analytics
- Automated data quality validation using Great Expectations

---

### 📈 [Real-Time Financial Transaction Streaming](https://github.com/saikrishna-suryavamsham/financial-streaming-pipeline)
> Live streaming pipeline for financial transactions with real-time Power BI observability dashboard

**Tech:** Apache Kafka · Spark Streaming · PostgreSQL · Power BI · Docker

- Python producer simulating **high-frequency transaction events** through Kafka topics
- Sub-second ingestion latency with Spark Streaming consumer and PostgreSQL sink
- Power BI dashboard tracking data freshness, processing latency, and anomaly alerts

---

### 💼 [HR Payroll Analytics Data Warehouse](https://github.com/saikrishna-suryavamsham/hr-payroll-dwh)
> Cloud data warehouse for HR and payroll analytics with dbt transformation layer

**Tech:** Python · Snowflake · dbt · Tableau · Great Expectations · Airflow

- Ingests payroll, benefits, and employee records from multiple HRIS source systems
- dbt models implementing star-schema dimensional design with automated tests
- Tableau dashboards for headcount trends, payroll summaries, and benefits utilization

---

### ☁️ [End-to-End Data Lakehouse](https://github.com/saikrishna-suryavamsham/aws-data-lakehouse)
> Multi-layer AWS data lakehouse architecture (Raw → Processed → Curated) with serverless query layer

**Tech:** AWS S3 · AWS Glue · Amazon Athena · Amazon QuickSight · Terraform

- Three-layer S3 architecture with Hive-style partitioning reducing data scan volume by **70%**
- Serverless AWS Glue ETL jobs with automated schema discovery
- Athena query layer + QuickSight dashboards for self-service analytics
- Infrastructure-as-Code with Terraform for reproducible deployments
